# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability, please send an email to your.email@example.com.

Please do NOT create a public GitHub issue for security vulnerabilities.

We will respond within 48 hours and work with you to resolve the issue.

## Security Best Practices

When using this library:

1. Never expose your AI API keys in client-side code
2. Always validate user input on the server side
3. Implement rate limiting to prevent abuse
4. Use HTTPS for all API communications
5. Keep dependencies up to date
6. Review the data adapter implementation for SQL injection risks
